# Следене на задачи

Българска система за следене на задачи, която замества Excel файл с по-удобен работен списък.

Приложението позволява въвеждане на задача с **рег. №**, **дата**, **срок** и автоматично изчислява **до кога трябва да бъде изпълнена задачата**.

## Функции

- Рег. № на задачата
- Дата на регистрация
- Автоматично изчисляване на краен срок
- Поддържани срокове:
  - 10 работни дни - изчакване на отговор
  - 14 работни дни
  - 20 работни дни
  - 30 календарни дни
- Статуси:
  - За изпълнение
  - В процес
  - За преглед
  - Готово
- Поле „Остава от задачата“, достъпно и редактируемо във всеки етап
- Отговорник
- Приоритет
- Етикети
- Търсене и филтри
- Табло със статистики
- Обновяване на дата и срок директно от картата на задачата
- Локално запазване на данните в браузъра / приложението
- Desktop версия чрез Electron
- GitHub Actions workflow за автоматично създаване на Windows `.exe`

## Важно за сроковете

Работните дни се смятат по български работен календар: събота, неделя, официални празници, Великденски празници и въведени допълнителни почивни дни не се броят като работни дни.

## Пускане като уеб приложение

Инсталирай Node.js 22 или по-нова версия.

След това изпълни:

```bash
npm install
npm run dev
```

Отвори адреса, който Vite показва в терминала. Обикновено е:

```txt
http://localhost:5173
```

## Пускане като desktop приложение по време на разработка

```bash
npm install
npm run electron:dev
```

## Създаване на Windows .exe локално

На Windows компютър изпълни:

```bash
npm install
npm run dist
```

Готовият installer ще бъде в папката:

```txt
release/
```

## Създаване на .exe през GitHub

В проекта има готов workflow:

```txt
.github/workflows/build-windows.yml
```

### Вариант 1: ръчно стартиране

1. Качи проекта в GitHub repository.
2. Отвори таб **Actions**.
3. Избери **Build Windows EXE**.
4. Натисни **Run workflow**.
5. След приключване изтегли файла от **Artifacts**.

### Вариант 2: чрез Git tag

```bash
git tag v1.0.0
git push origin v1.0.0
```

GitHub автоматично ще създаде Windows `.exe` като artifact.

## Изчисляване на срокове по български работен календар

Версията включва изчисляване на срокове с:

- събота и неделя като почивни дни;
- официалните празници в България;
- Великденските празници според православния календар;
- неприсъствени дни, когато официален празник без Великден се падне в събота или неделя;
- еднократни допълнителни почивни дни, добавени в кода.

При срок **10 работни дни - изчакване на отговор**, **14 работни дни** и **20 работни дни** системата брои само работни дни. При срок **30 календарни дни** се добавят 30 календарни дни, но ако крайната дата се падне почивен ден, тя се измества до следващия работен ден.

Еднократните почивни дни се намират в `src/main.tsx` в масива `EXTRA_NON_WORKING_DAYS`.
